# ⭐ Stars & 👁️ Watch

## ⭐ Stars

### What it does
Starring a repo is like bookmarking it. It shows appreciation for the project and saves it to your starred repos list for easy access later.

### How to Star
Click the **⭐ Star** button on any repo page.

### Where to find starred repos
`Your profile` → **Stars** tab
or: `https://github.com/your-username?tab=stars`

### Why Stars Matter
| For Users | For Repo Owners |
|---|---|
| Bookmark interesting repos | Social proof of popularity |
| Find them quickly later | Appears higher in search |
| Build a personal library | Shows community interest |

## 👁️ Watch

### What it does
Watching a repo subscribes you to notifications for activity — new issues, pull requests, releases, commits, and discussions.

### Watch Options

| Option | What you get notified about |
|---|---|
| **Not watching** | Only when mentioned or participating |
| **Watching** | All activity (issues, PRs, commits) |
| **Releases only** | New releases/tags only |
| **Ignoring** | No notifications at all |
| **Custom** | Choose specific event types |

### How to Watch
Click the **👁️ Watch** dropdown → choose your preference.

---

## 🔔 Managing Notifications

Too many notifications? Control them at:
`GitHub → Settings → Notifications`

Options:
- Email vs web notifications
- Per-repo overrides
- Filter by reason (assigned, mentioned, subscribed)

### Quick tip — Unwatch noisy repos:
```
Your profile → Settings → Notifications → Subscriptions
→ Filter and unwatch repos sending too many notifications
```
